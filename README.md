# Liquid Color Sort

> A native Android puzzle game in the liquid-sorting genre. Built with Kotlin, Jetpack libraries, AdMob monetization, and a deterministic level generator. Targeting Google Play release in ~14 weeks.

## Tech Stack

| Layer | Technology |
|---|---|
| Language | Kotlin 2.0 |
| Min SDK | API 24 (Android 7.0) |
| Build | AGP 8.4, Gradle 8.9, version catalog |
| DI | Hilt 2.51 |
| Navigation | Jetpack Navigation + Safe Args |
| Persistence | Room 2.6, DataStore 1.1 |
| Monetization | AdMob 23.x (banner, interstitial, rewarded) |
| Game Rendering | Custom Canvas `TubeView` + `ValueAnimator` |
| Testing | JUnit 5, MockK, Robolectric |
| CI | GitHub Actions |

## Architecture

```
Single Activity (MainActivity)
    ↓ NavController
  ┌─────────────────────────────────────────────────────┐
  │  HomeFragment   LevelSelectFragment   GameFragment  │
  │  HomeViewModel  LevelSelectViewModel  GameViewModel │
  └─────────────────────────────────────────────────────┘
         ↓                   ↓                  ↓
  core/engine/       core/level/         data/repository/
  MoveValidator      LevelRepository     ProgressRepository
  SolverHint         LevelGenerator      SettingsRepository
         ↓                                      ↓
  core/model/                            data/local/
  GameState                              AppDatabase (Room)
  Tube                                   SettingsDataStore
  LiquidColor
```

### Key design rules
- **Single source of truth**: `GameViewModel` holds the only mutable `GameState`. Views never mutate state.
- **Input locking**: `isAnimating` flag in `GameViewModel` is set by the Fragment before animation starts and cleared after — preventing any tap from corrupting state mid-animation.
- **Immutable model**: every move returns a new `GameState`; history is a list of past tube snapshots.
- **Deterministic levels**: `LevelGenerator` uses a fixed seed — the same seed always produces the same level.

## Building

### Prerequisites
- JDK 17+
- Android Studio Iguana or later (recommended)
- Android SDK API 35

### Command line
```bash
# Run unit tests
./gradlew test

# Assemble debug APK
./gradlew assembleDebug

# Lint
./gradlew lint
```

### Before release
1. Replace AdMob placeholder IDs in `app/build.gradle.kts` release block
2. Create `app/release.properties` with keystore credentials
3. Add real `google-services.json` (if using Crashlytics)

## Level System

- **Levels 1–50**: Hand-authored in `app/src/main/assets/levels.json`
  - 3 colors / 5 tubes → 12 colors / 14 tubes
  - Verified by construction (each color appears exactly `capacity` times)
- **Levels 51–100**: Generated by `LevelGenerator` with fixed seeds
  - BFS solvability verified on every generated level
  - Same seed → same level (determinism contract)

## Ad Strategy

| Ad Type | Placement |
|---|---|
| Banner | Home, Level Select |
| Interstitial | Every 3 level completions |
| Rewarded | User-initiated hint (BFS solver move) |

Test ad unit IDs are used in debug builds. Replace with real IDs before publishing.

## Monetization Stretch: Remove Ads IAP

The `PurchaseManager` interface and `StubPurchaseManager` no-op are scaffolded. To activate full billing:
1. Add `com.android.billingclient:billing-ktx` to dependencies
2. Implement `BillingManagerImpl`
3. Bind via a Hilt `BillingModule`
